# 📋 Cube Castle 项目文档管理规范

> **版本**: v1.0  
> **生效日期**: 2025年7月31日  
> **适用范围**: Cube Castle 项目所有文档  
> **维护者**: 项目架构团队

## 🎯 文档管理目标

### 核心原则
- **标准化**: 统一的格式和结构规范
- **可维护性**: 易于更新和维护的文档组织
- **可发现性**: 清晰的分类和命名规范
- **版本控制**: 完整的版本历史和变更追踪
- **国际化**: 支持中英文双语文档

### 质量标准
- **准确性**: 文档内容与代码实现保持同步
- **完整性**: 覆盖所有必要的技术和业务信息
- **清晰性**: 易于理解的语言和结构
- **时效性**: 及时更新和维护

## 📝 文档格式规范

### Markdown 标准

#### 标题层级
```markdown
# 一级标题 (文档标题)
## 二级标题 (主要章节)
### 三级标题 (小节)
#### 四级标题 (细分内容)
```

#### 文档头部信息
```markdown
# 文档标题

> **版本**: v{x.y.z} | **更新日期**: YYYY-MM-DD | **状态**: {状态} | **作者**: {作者}

## 概述
[文档概述内容]
```

#### 代码块格式
```markdown
# 行内代码
使用 `反引号` 包围行内代码

# 代码块
```language
代码内容
```

# 配置文件示例
```yaml
key: value
nested:
  item: value
```
```

#### 表格格式
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
| 值4 | 值5 | 值6 |
```

#### 链接和引用
```markdown
# 内部链接
[链接文本](../path/to/document.md)

# 外部链接
[链接文本](https://example.com)

# 锚点链接
[跳转到章节](#章节标题)
```

#### 状态标识
```markdown
✅ 已完成    🔄 进行中    ❌ 未开始    ⚠️ 需要注意
🆕 新增功能  🔧 修复      📈 改进      📊 数据
🚀 发布      🔒 安全      📚 文档      🧪 测试
```

### 图片和图表规范

#### 图片命名和存储
- **命名格式**: `{document_name}_{figure_type}_{number}.{ext}`
- **存储位置**: `docs/assets/images/{category}/`
- **支持格式**: PNG, JPG, SVG
- **分辨率**: 最低1080p，推荐使用SVG矢量图

#### 图表类型
- **架构图**: 使用统一的图标库和配色方案
- **流程图**: 遵循标准的流程图符号
- **时序图**: 使用Mermaid或PlantUML格式
- **数据图表**: 使用一致的配色和样式

### 文档审核流程

#### 创建阶段
1. **起草**: 作者创建文档草案
2. **自检**: 作者进行格式和内容自检
3. **提交**: 提交PR进行代码审核

#### 审核阶段
1. **技术审核**: 技术负责人审核技术准确性
2. **格式审核**: 文档管理员审核格式规范
3. **最终审核**: 项目负责人最终审批

#### 发布阶段
1. **合并**: 合并到主分支
2. **发布**: 更新文档索引
3. **通知**: 通知相关团队成员

### 文档维护规范

#### 定期审核
- **月度审核**: 检查文档准确性和时效性
- **季度审核**: 评估文档结构和完整性
- **年度审核**: 归档过时文档，更新文档规范

#### 废弃流程
1. **标记废弃**: 在文档头部添加废弃标记
2. **设置期限**: 设定废弃删除时间
3. **通知相关方**: 通知使用该文档的团队
4. **归档处理**: 移动到archive目录

- **主要语言**: 中文 (zh-CN)

- **更新同步**: 确保不同语言版本的同步更新

### 自动化流程

#### CI/CD集成
```yaml
# .github/workflows/docs.yml
name: Documentation
on:
  pull_request:
    paths: ['docs/**']
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Lint Markdown
        uses: markdownlint/markdownlint-action@v1
      - name: Check Links
        uses: gaurav-nelson/github-action-markdown-link-check@v1
```

#### 文档生成
- **自动目录**: 生成文档目录索引
- **API文档**: 从代码注释自动生成API文档

## 📊 质量控制

### 质量检查清单

#### 内容质量
- [ ] 信息准确性
- [ ] 逻辑完整性
- [ ] 示例可运行性
- [ ] 链接有效性

#### 格式质量

#### 结构质量

- [ ] 版本信息完整
- [ ] 索引关系正确

### 评审标准

#### 评分标准 (1-5分)
- **内容准确性**: 技术信息是否准确无误
- **结构清晰性**: 文档结构是否逻辑清晰
- **可用性**: 是否易于理解和使用
- **完整性**: 是否覆盖所需的所有信息
- **时效性**: 信息是否保持最新状态

#### 通过标准
- 所有项目平均分 ≥ 4.0
- 无严重错误或遗漏
- 符合格式规范要求
