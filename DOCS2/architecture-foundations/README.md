## 📋 概述

本目录包含项目的最高级别架构指导文档，定义了系统的哲学基础、技术方法和战略方向。这些文档作为项目的"宪法"级文档，为所有开发工作提供根本性指导。

## 🏰 核心架构文档

### 📜 基础规约文档
- **[元合约v6.0规范](metacontract-v6.0-specification.md)** - 项目最高技术宪章
  - **重要性**: 最高级别 - 作为平台开发的唯一法律依据
  - **文档类型**: 技术规约 - 机器可读的架构护栏
  - **适用范围**: 全平台开发、治理和演进工作
  - **核心内容**: 
    - 城堡模型定义和四大支柱实现
    - 完整的API规约和数据结构定义
    - AI治理模型和对话式交互框架
    - 执行保障与合规性规约
  
- **[城堡蓝图](castle-blueprint.md)** - 战略架构实现路径
  - **重要性**: 最高级别 - 系统哲学和技术方法定义
  - **文档类型**: 战略指导 - 架构决策的哲学基础
  - **适用范围**: 架构设计、团队决策、长期演进规划
  - **核心内容**:
    - "雄伟单体"与"城堡模式"的战略选择
    - 模块化单体的详细设计原则
    - 垂直切片执行策略和演进路径
    - "绞杀者无花果"长期演进规划

- **[CQRS统一架构实施指南](cqrs-unified-implementation-guide.md)** - CQRS架构的统一实施标准
  - **重要性**: 最高级别 - 所有模块CQRS实施的唯一技术依据
  - **文档类型**: 技术规约 - CQRS架构实施的宪法级文档
  - **适用范围**: 所有业务模块的CQRS架构迁移和实施
  - **核心内容**:
    - CQRS架构五大宪法原则和城堡模型契合
    - 命令端、查询端、事件驱动的统一实现标准
    - 三阶段城堡化迁移战略和质量保证宪章
    - 风险管理、性能优化和运维监控标准

## 🔄 文档演进与治理

### 变更管理
- **宪法级文档变更**: 需要架构委员会全体一致同意
- **版本控制**: 采用语义化版本控制 (Semantic Versioning)
- **向后兼容**: 重大变更需要提供迁移路径

### 审核机制
- **定期审核**: 每季度审核一次文档的时效性和准确性
- **变更审核**: 所有变更都需要通过GitHub Pull Request进行审核
- **影响评估**: 重大变更需要进行全面的影响评估

## 🎯 使用指南

### 对于架构师
1. **设计决策**: 参考城堡蓝图的战略原则和设计哲学
2. **技术选型**: 以元合约v6.0的技术规约为准
3. **演进规划**: 遵循文档中定义的演进路径和触发器

### 对于开发团队
1. **开发规范**: 严格遵循元合约v6.0中的技术规约
2. **模块设计**: 按照城堡模型的定义进行模块划分
3. **API设计**: 遵循元合约中定义的API设计模式

### 对于项目管理
1. **里程碑规划**: 参考垂直切片执行策略
2. **风险管理**: 重点关注文档中识别的关键风险点
3. **资源配置**: 根据文档指导合理配置团队资源
